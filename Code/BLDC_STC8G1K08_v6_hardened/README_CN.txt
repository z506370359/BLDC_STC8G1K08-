【项目简介】

本项目把原 PIC16F1503 单霍尔BLDC风扇控制器方案移植到 STC8G1K08-38I-TSSOP20,
硬件采用 P-MOS 上管 (NCE60P25K) + N-MOS 下管 (NCE6020AK) 互补半桥拓扑,
支持 5~18V 宽电压输入, 2A 以下工作电流, 典型应用为 12V 散热风扇。

相比原版, 功能保持不变 (软启动 / 闭环PI / FG反馈 / 堵转保护), 并修复了
多个移植过程中遇到的致命bug, 代码适配硬件 PWM 极性反转特性。


【文件清单】

  stc8g.h          - STC8G寄存器定义 (SFR地址 + 位定义)
  bldc_config.h    - 项目配置 (引脚分配 + 参数常量 + 宏定义)
  main.c           - 主程序 (状态机 / ISR / PI控制 / ADC)
  README_CN.txt    - 本文件

【核心技术点】

1. PCA 8位PWM驱动下桥臂N-MOS, 频率23.4kHz (高于20kHz避免啸叫)

2. 双沿INT0触发霍尔换相, 实现2步换相逻辑

3. 超前角控制: 霍尔边沿延迟200μs后才执行换相, 补偿位置偏差

4. 软切换: 下次换相前500μs自动降低占空比到一半, 减少换相冲击

5. 32位电气周期测量: 由4个半周期累加得到, 平均化抑制抖动

6. PI闭环转速控制: 每50ms运算一次, 支持±105%范围

7. 软启动: 占空比从20%线性爬升到30%, 历时1500ms, 避免启动电流冲击

8. 堵转保护: 硬件过流中断 + 软件超时(32.7ms无换相)双重保护

9. ADC非阻塞采样: 每1ms触发, 25次平均(25ms平滑滤波)

10. 多级故障保护: 单次堵转→5秒重试, 连续5次→永久停机(ST_FAULT)


【状态机流程 (v6新增ST_FAULT)】

            上电
             ↓
      ST_MOTOR_OFF ←──────────────────┐
       (等待输入)                      │
             ↓ (输入>3%持续200ms)       │
      ST_SOFT_START                    │ (5秒后)
       (占空比爬升)                    │
             ↓ (转速达标或超时)         │
      ST_MOTOR_RUN ──堵转──→ ST_MOTOR_RESTART
       (PI闭环控制)           │
             │                 │ (堵转计数<5)
             │                 ↓
             │          [回到OFF重启]
             │                 │
             │                 │ (堵转计数>=5)
             │                 ↓
             │          ST_FAULT
             │         (永久停机, 等待输入归零)
             │                 │
             │                 │ (用户输入<=3%)
             │                 ↓
             └──── 任何运行状态 ──→ 清零计数回到 ST_MOTOR_OFF


【修复历史】

v1.0 首次STC移植 (带有多个致命bug)
v2.0 修复6项致命问题:
     - ADC右对齐拼接错误 (导致永远满转速)
     - GetSpeed读32位变量的原子性
     - 关键变量volatile修饰符缺失
     - Timer0中断内ADC轮询阻塞
     - FAN_OFF_TO_START_DUTY=0的恒假bug
     - P3.2高阻态与开漏霍尔IC不兼容
v3.0 修复2项:
     - 软切换标志位未清零, 导致PCA中断连续减半至0
     - Timer1飞读问题, 用双读校验解决
v4.0 修复1项致命:
     - PCA PWM极性反转 (CCAPnH=0实际为100%!), 导致PI正反馈必定堵转
v5.0 适配新硬件:
     - 互补半桥拓扑 (P-MOS+N-MOS)
     - FG输出经NPN反相, SetFgRdLdHigh/Low宏需反转极性
     - P3.6/P3.7外部分压器替代PIC内部DAC基准
v6.0 代码加固:
     - 新增 ReadMsClk() 原子读取宏, 消除u16撕裂读风险
     - 新增 ST_FAULT 永久故障状态, 避免硬故障下无限重启循环
     - 连续堵转 FAULT_LOCK_THRESHOLD(默认5)次后进入永久停机
     - 正常运行 LOCK_CNT_RESET_TIME(默认10秒) 后自动清零堵转计数

【重要提示 (烧录前必读)】

1. 引脚分配必须与 bldc_config.h 一致, 特别是:
   - P1.0/P1.1 必须是 CCP1/CCP0 (硬件PWM引脚, 不可更换)
   - P3.2 必须是 INT0 (霍尔中断)
   - P3.6/P3.7 必须是 CMP-/CMP+ (比较器)

2. PCA PWM极性反转处理已在代码中完成, 使用 WritePwmDuty() 接口时
   直接传入逻辑占空比 (0=关断, 255=满), 不要直接改 CCAPnH

3. P3.2默认配为准双向口(内部弱上拉), 如果外部已有上拉可改为高阻(10)

4. FG反馈极性: 本代码默认FAN_BACK_OUT=0 (FG模式, NPN反相)
   如果您的电路FG是MCU直出, 改为: SetFgRdLdHigh()=PIN_FG_RD=1

5. 首次通电前务必检查过流基准值是否合理:
   Vref = 5V × R22/(R21+R22) = 5V × 4.7K/(47K+4.7K) ≈ 0.455V
   Rsense = 采样电阻并联值 (例: 200mΩ×2并联 = 100mΩ)
   Ioc = Vref / Rsense (应合理匹配电机工作电流, 典型为3~5倍额定)


【调试建议】

如果烧录后发现异常:

  1. 电机不转:
     - 测量P1.0/P1.1是否有PWM波形 (23.4kHz)
     - 测量P3.2是否有霍尔方波信号
     - 确认上桥臂NPN和P-MOS栅极电压关系

  2. 电机转但抖动大:
     - 检查采样电阻SEN信号, 可能过流误触发
     - 降低PWM_P_PARAMETER (比例增益), 如从12降到8
     - 检查霍尔信号完整性, 走线是否过长或受干扰

  3. 速度无法调节:
     - 测P1.4 ADC输入电压是否随PWM/电压变化
     - 检查R15+C10滤波网络参数

  4. 启动频繁失败:
     - 增大 SOFT_START_END_DUTY (从30%提高到40%)
     - 增大 SOFT_START_DELAY (从1500ms提高到2000ms)
     - 检查电源容量是否不足以支持启动电流
